Kotlin Nullable Types

Kotlin has datatypes designed specifically for nullability. These are types that can hold null references. To declare a nullable type, you add a question mark to the type during declaration:

var myNullableString:String? = null
var myNullableInt:Int? = null
var myNullableLong:Long? = null

Safe Call Operator(?.)

The safe call operator is used to access nullable variables safely. say we have a nullable String called firstName and you want to get its length property. The verbose way to check whether it is null would be:

if(firstName!=null){
   return firstName.length
}else{
   return null
}

Using the Safe Call operator, all we have to do is call firstName?.length

Elvis Operator(?:)

The Elvis operator is used to provide a default value in case an object is null. using our previous example, letâ€™s say we need to access the firstName variable, the verbose way would be to check whether it is not null then return its value or return the default value if it is null:

   return firstName
}else{
   return 'Empty Name'
}

Using the Elvis operator, all we have to do is call firstName?:"Empty Name" .
The Elvis operator can be used in combination with the safe call operator, so that if a variable is null, it returns a default value:

var nameLength = firstName?.length ?: 0 //returns 0 if firstName is null

Lateinit Modifier

As its name suggests, the lateinit keyword is used with non-null objects that cannot be initialized at declaration time(Think of some of Androidâ€™s object properties that are initialized using special methods like onCreate() or situations where you are using Dependency Injection).

In Kotlin, non-null properties must be initialized with non-null values at declaration time. The lateinit modifier allows us to get away with not initializing these properties.

Accessing a lateinit property that has not been initialized however will throw an exception, UninitializedPropertyAccessException so you should check whether the property has been initialized first before trying to access it using propertyName.isInitialized.
